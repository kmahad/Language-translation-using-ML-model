1. Abstract & Introduction

Neural Machine Translation (NMT) represents a major advancement in the field of Natural Language Processing. This project focuses on building, training, and deploying a custom Sequence-to-Sequence (Seq2Seq) translator based on the original Transformer Architecture proposed by Vaswani et al. (2017).

Traditional translation pipelines relied heavily on Recurrent Neural Networks (RNNs) and Long Short-Term Memory (LSTM) networks. However, these architectures struggle with parallelization due to their sequential design and suffer from information loss when translating long-range sentences. The Transformer architecture solves these limitations by utilizing the Self-Attention Mechanism, allowing the model to focus on relevant context words across the entire sequence simultaneously, regardless of their distance.

Key Objectives: Implementing PyTorch encoder-decoder layers from scratch, compiling a SentencePiece subword tokenizer, configuring a responsive training monitoring dashboard, and tuning inference with beam search and repetition penalties.

2. System Architecture

The model architecture consists of a symmetric encoder-decoder pipeline. The encoder maps an input sequence of symbol representations to a sequence of continuous representations, which the decoder then translates into an output sequence token-by-token.

3.2 Embedded Mathematics

To provide position awareness without using recurrence, Sinusoidal Positional Encoding is added directly to token embeddings: 

PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

In the self-attention block, queries (Q), keys (K), and values (V) are projected, split across heads, and computed using Scaled Dot-Product Attention: 

Attention(Q, K, V) = softmax( (Q * K^T) / sqrt(d_k) ) * V

4. Advanced Decoders & Project Optimization

During integration testing, two critical bugs were diagnosed and resolved to stabilize the pipeline for college showcase:

4.1 Weight Tying & Embedding Scale Normalization

We linked target embeddings to output projection parameters to save network capacity (Weight Tying). Because output logits are dot products, initializing raw embeddings with standard normal distribution caused high variance outputs (~22.5 std), inflating initial cross-entropy loss to 90+. We resolved this by initializing embedding weights with $N(0, d_{model}^{-0.5})$ to ensure unit variance and stable initial training loss (~10.19).

4.2 Repetition Penalization and Max Length Safety Capping

In early training epochs, models tend to loop words. We implemented a Hugging Face-style repetition penalty inside the decoding script: 

# Apply repetition penalty dynamically during generation
if repetition_penalty != 1.0:
    for token in set(generated_tokens):
        if logits[0, token] > 0:
            logits[0, token] /= repetition_penalty
        else:
            logits[0, token] *= repetition_penalty

Additionally, because sequence length was optimized to 64 for demonstration speed, we created an active capping handler: max_len = min(max_decode_len, model_pe_limit). This successfully prevents indexing exceptions at generation boundaries.

5. Visualization Dashboard & Performance

The project contains a comprehensive Flask-based web-control panel serving as the visual interface. It gives users immediate, visual control over every aspect of the pipeline:

  • Interactive Translation Pane: Translate input text using either Greedy Decoding or Beam Search ($k=4, 8$) with active repetition penalties.
  • Pipeline Task Control: Start/abort dataset preprocessing, training runs, or evaluation tasks, and view standard logs inside a real-time streaming output console.
  • Loss Progress Tracking: Live graphing of training and validation loss updates as epochs progress.
  • Config Editor: Live adjustments to model parameters directly from the browser.
